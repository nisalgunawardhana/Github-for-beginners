# Issue #2: Practice Git Commands

**Issue Type:** Task  
**Priority:** Medium  
**Labels:** good first issue, practice, beginner-friendly

---

## Description

Practice essential Git commands by adding a new section to the `practice-file.md`. This exercise will help you understand the complete Git workflow.

---

## Objectives

- [ ] Create a feature branch
- [ ] Make meaningful commits
- [ ] Practice staging changes
- [ ] Push changes to remote repository
- [ ] Create a pull request

---

## Steps to Complete

1. Ensure your fork is up to date with the main repository
2. Create a new branch: `feature/add-new-section`
3. Open `practice-file.md`
4. Add a new section about your favorite coding topic or technology
5. Make at least 2-3 separate commits showing your progress
6. Push your branch to GitHub
7. Create a pull request with a clear description

---

## Required Changes

Add a new section to `practice-file.md` that includes:
- A topic title (e.g., "Web Development", "Data Science", etc.)
- 3-5 bullet points about the topic
- Any useful tips or resources

---

## Git Commands You'll Practice

```bash
git checkout -b feature/add-new-section
git add practice-file.md
git commit -m "Your descriptive message"
git push origin feature/add-new-section
```

---

## Success Criteria

- ✅ Branch created with appropriate name
- ✅ Changes made to practice-file.md
- ✅ Multiple descriptive commits made
- ✅ Changes pushed to GitHub
- ✅ Pull request created
